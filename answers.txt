Q1: In your own words, explain what happens when you type a website address like google.com and press Enter.
When I type google.com into my browser and press Enter, the first step is a Domain Name System lookup, where DNS resolves the domain name into an IP address so the browser knows where to send the request.

The request is then sent to a load balancer, which distributes traffic across different web servers using algorithms such as round-robin to improve performance and prevent overload.

The web server receives the request and processes it. If the request requires only static content such as images, HTML, or files, the web server responds directly. However, if additional processing is needed, the request is forwarded to the application server.

The application server handles the business logic, runs the required code, and communicates with the database when information needs to be retrieved or stored. The response is then sent back through the web server and load balancer to the browser, where the webpage is displayed.

Client is any device that can be used to send request and receive it back. Example, your phone using WhatsApp to send a message 
Whilst 
A server is a device that provides services data or information to the client when they request them. Example, the WhatsApp server that stores and delivers message to the receiver.

Q3:What is DNS and why do we need it?
 DNS (Domain Name System) converts website names like google.com into IP addresses that computers understand. We need it so users can easily access websites without remembering long IP addresses of every website we want to visit
Q4: A firewall and a load balancer are both part of how the web works. What does each one do? How are they different?
A firewall is a network security device that monitors and controls incoming and outgoing traffic to protect networks from unauthorized access and cyber threats.
Whilst
A load balancer helps distributes traffic from incoming requests across multiple servers to prevent overload and improve performance. The difference is that a firewall focuses on security, while a load balancer focuses on managing traffic and keeping services running smoothly.
